# 운동기록 웹앱 — 설계 문서

작성일: 2026-06-22
상태: 설계 승인됨 (구현 전)

## 1. 목적 / 문제

헬스장에서 운동 전 "오늘 어떤 기구를 무슨 무게·휴식으로 할지"를 매번 **메모장**에 적던 불편을 없앤다.
시중 앱은 무료로 쓸 만한 게 없음 → **나 혼자 쓸 무료 개인 앱**을 직접 만든다.

핵심 가치 = (1) 루틴 미리 짜기, (2) 운동 중 세트별 무게/반복 기록 + 휴식 타이머 자동, (3) 지난 기록 조회("저번엔 몇 kg?").
부가로 (4) 가벼운 대시보드(나중 슬라이스).

## 2. 범위

### 포함 (v1)
- 운동(기구) 라이브러리 — 사용자가 직접 추가 (빈 상태 시작, 기본목록 없음)
- 루틴(템플릿) 만들기/편집 — 운동 묶음 + 목표 세트수·휴식시간
- 운동 세션 진행 — 루틴 따라가기 + 자유 추가, 세트별 무게(kg)/반복 입력
- 휴식 타이머 — 세트 완료 시 자동 시작, 끝나면 소리 + 진동(햅틱) 알림
- 기록 조회 — 지난 세션 보기, 기구별 "저번 무게" 빠르게 확인

### 나중 슬라이스
- 대시보드 — 총 볼륨, 자주 한 운동, 무게 추이 등 가벼운 통계 ("있으면 좋음")

### 제외 (안 함)
- 습관/루틴 체크(물·걸음 등) — 불필요
- 식단/칼로리 추적
- 계정·로그인·서버·동기화 (혼자 씀, 로컬 저장)
- 기본 운동 목록 번들 (직접 추가가 더 단순)

## 3. 기술 스택 / 플랫폼

- **웹앱(PWA)**. 바닐라 JS ES 모듈, **빌드 도구 0**. localStorage 영구 저장.
- 혼자 사용 → 계정·서버·앱스토어·비용 전부 불필요.
- 아이폰에서 Safari → "홈 화면에 추가"하면 전체화면 앱처럼 사용. (사용자 PC=Windows, 폰=아이폰 → 네이티브 iOS 빌드는 Mac 필요라 PWA가 최적.)
- iOS PWA 한계: 화면 꺼짐/백그라운드 알림 제한적. 실사용(헬스장서 화면 켜둠)엔 문제없음 — 타이머는 앱 활성 상태 기준.
- 모듈 검증: 순수 로직은 `node --test`, DOM은 로컬 http 서버 + 헤드리스 브라우저.

## 4. 화면 구성 (3 + 1)

1. **루틴 화면 (홈)** — 내가 만든 루틴 목록. 탭 → 그 루틴으로 세션 시작. 루틴 추가/편집 진입.
2. **운동 중 화면** — 현재 기구 + 세트별 무게/반복 입력. "세트 완료" → 휴식 타이머 링 자동 시작 + 끝나면 소리/진동. 다음 기구로 이동.
3. **기록 화면** — 지난 세션 리스트. 기구 선택 시 과거 무게/반복 이력("저번 무게").
4. **대시보드** (나중) — 가벼운 통계.

## 5. 데이터 모델 (localStorage)

계획(운동·루틴)과 실제 기록(세션)을 **분리**한다 → 조회·통계가 깔끔.

```
운동(Exercise)  : { id, 이름, 종류('머신'|'덤벨'|'케이블'|'기타'), 기본휴식초 }
   - 사용자가 만든 기구 라이브러리. 빈 상태 시작.

루틴(Routine)   : { id, 이름, 운동목록[ { 운동id, 목표세트수, 목표휴식초 } ] }
   - 예: "가슴날" = [랫풀다운×3, 체스트프레스×3, 케이블플라이×3]

세션(Session)   : { id, 날짜, 루틴id, 기록[ { 운동id, 세트[ { 무게kg, 반복 } ] } ] }
   - 운동 한 번 = 세션 하나. 기록 화면·대시보드가 이걸 읽음.
```

저장: localStorage에 JSON 직렬화. 키 구조와 마이그레이션은 구현 계획에서 확정.

## 6. 아키텍처 규칙

- **로직과 렌더 분리.** 계산(타이머·통계·이력 조회)은 순수 로직 모듈에, DOM 렌더는 UI 모듈에. UI는 계산 금지.
- 파일은 한 가지 책임. 커지면 쪼갠다.
- 밸런스/설정 숫자는 명명 상수 (기본 휴식초 등).
- 순수 로직은 `node --test` 단위 테스트. TDD: 실패 테스트 → 최소 구현 → 통과 → 커밋.

## 7. 네이티브 느낌 디자인 규칙 (필수 — 모든 화면 적용)

화풍 = **"미드나잇"**: 다크 배경(#0d0f14 계열) + 형광 그린 액센트(#00e599). 스포티, 숫자 큼, 강렬.

- 애니메이션은 `transform`/`opacity`만 (60fps, 레이아웃 리플로우 금지)
- 시스템 폰트 스택, 최소 44px 터치영역, 일관된 여백 간격(8의 배수)
- 탭 하이라이트·텍스트 선택 제거, 관성 스크롤
- 세트 완료·타이머 종료에 햅틱(`navigator.vibrate`) + 부드러운 전환
- PWA 전체화면(manifest `display: standalone`), 다크 테마 고정
- 색·간격·둥글기·그림자 = **디자인 토큰**(CSS 변수)으로 통일 → 들쭉날쭉 방지

## 8. 에셋 (옵션)

- 핵심 화풍은 픽셀아트 아님(다크 스포티 UI).
- **앱 아이콘**, **빈 화면 일러스트** 정도는 Retro Diffusion(RD_PRO)으로 만들어도 됨 — 필수 아님.

## 9. 성공 기준

- 헬스장에서 폰으로 루틴 골라 → 세트 기록 → 타이머 자동 → 다음 기구, 끊김 없이 한 흐름으로 됨.
- 기구 선택 시 "저번 무게"가 즉시 보임.
- 메모장보다 빠르고 깔끔. 디자인이 "예쁘다"고 느껴짐.
- 오프라인 동작, 데이터 영구 보존.

## 10. 미해결 (구현 계획에서 확정)

- localStorage 키 구조 / 향후 마이그레이션 방식
- 운동 중 화면에서 "자유 추가"(루틴에 없는 기구 즉석 추가) UX 디테일
- 대시보드 통계 항목 구체화(나중 슬라이스)
- PWA manifest / 서비스워커(오프라인 캐시) 범위
